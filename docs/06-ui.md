# 06 — UI 设计

---

## 6.1 整体 Layout

**Desktop**

- 固定左侧 Sidebar（宽 240px，不可收起）
- Sidebar 内容：应用 Logo / 名称（顶部）、导航菜单（Chat、Knowledge、History、Quality、Settings）
- 右侧主内容区自适应宽度

**Mobile**

- 全屏内容区
- 底部固定 Tab Bar：Chat、Knowledge、History、Quality、Settings（5 个 Tab）

---

## 6.2 `/chat` — FAQ 问答

**交互约定：**

- 进入 `/chat` 时若无活跃会话，呈现空消息列表（pending 状态，`conversationId = null`）
- 用户发送第一条消息时，前端自动调用 `POST /conversations` 创建会话，URL 更新为 `/chat/:id`
- "新建对话"操作仅清空消息列表并重置 `conversationId = null`，不调用 API

**Desktop**

- 主内容区上下分割：消息列表（滚动）+ 底部固定输入区
- 输入区：多行文本框 + 发送按钮（Enter 发送，Shift+Enter 换行）
- user 消息：右对齐气泡
- assistant 消息：左对齐气泡，消息下方两个折叠区：
  - `[引用来源 ▾]`：展开后列出来源文档名 + 段落内容摘要
  - `[评估中... ⟳]` / `[✓ F:0.92  AR:0.87  CP:0.79 ▾]`：展开后显示各指标分数和 reason（仅在线评估开启时显示）
- 消息右下角：👍 / 👎 反馈按钮

**Mobile**

- 同 Desktop，引用来源和评估结果内联折叠在消息卡片内，全屏展示

---

## 6.3 `/knowledge` — 知识库管理

**Desktop**

- 顶部上传区：拖拽 / 点击选择文件（支持 TXT、MD，单文件 ≤ 10MB）
- 文档列表（表格）：文件名、格式、状态 badge、上传时间、处理完成时间、操作列（删除 / 重试）
- 状态 badge：`pending`（灰）/ `processing`（蓝，带旋转动画）/ `done`（绿）/ `failed`（红）
- 处理中状态自动轮询刷新（3 秒间隔）

**Mobile**

- 顶部上传按钮（点击调起文件选择器）
- 卡片列表替代表格，状态 badge 展示在卡片右上角

---

## 6.4 `/history` — 历史记录

**Desktop**

- Master-detail 布局：左侧会话列表（宽 280px，固定）+ 右侧对话详情
- 左侧列表项：会话标题、创建时间，点击切换详情
- 右侧：完整对话消息流，含引用来源和评估结果（只读，不可输入）

**Mobile**

- 两级路由：`/history`（会话列表）→ `/history/:id`（对话详情）
- 详情页顶部有返回按钮

---

## 6.5 `/settings` — 系统设置

内部三个 Tab：LLM、Indexing、Query

**LLM Tab**（`/settings/llm`）

- Provider 选择（DeepSeek / OpenAI）
- Model 输入框
- Base URL 输入框
- Prompt 模板列表：名称、激活状态 badge、编辑 / 删除按钮
- 新建模板按钮（弹出编辑器，支持写 system prompt 内容）

**Indexing Tab**（`/settings/indexing`）

- 顶部提示条（始终显示）：「修改以下配置后，已有文档需重新 Embedding 才能生效」
- Chunking 策略选择（Fixed / Semantic）
- Chunk size 数字输入
- Chunk overlap 数字输入
- 保存后若有变更，页面额外弹出确认提示条

**Query Tab**（`/settings/query`）

- HyDE 开关（Toggle）
- Re-ranking 开关（Toggle）
- 在线评估开关（Toggle）
- Top-K 数字输入
- 对话历史窗口数字输入

**Desktop**：Tab 在内容区顶部水平排列
**Mobile**：`/settings` 展示 Tab 列表 → 点击进入子页面

---

## 6.6 `/quality` — 质量验证

内部两个 Tab：在线评估、离线评估

**在线评估 Tab**（`/quality/online`）

- 按会话分组列出评估记录，展示各消息三项指标分数汇总
- 点击消息展开：
  - 检索到的 chunks（内容 + 相似度分数 + rerank 分数）
  - 完整送入 LLM 的 prompt
  - 各指标分数和 reason

**离线评估 Tab**（`/quality/offline`）

- 显示当前测试集文件名和问题数量
- 运行说明：展示 CLI 命令供用户手动执行（`python eval/evaluate.py`）
- 测试集 JSON 格式说明（`[{ "question": "...", "expected_answer": "..." }]`）

**Desktop / Mobile** 布局同 Settings Tab 模式。
