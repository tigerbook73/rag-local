# 09 — Quality Validation 需求场景

---

## 为什么需要 Quality Validation？

RAG 系统的回答质量难以直观判断：答案听起来合理，但可能引用了错误的段落，或者与知识库内容不一致。Quality Validation 提供两种互补的评估手段，帮助发现问题并持续改进。

---

## 场景一：在线评估（对每次问答打分）

**使用者**：日常使用 RAG 系统的用户

**触发方式**：在 Settings 中开启 Online Evaluation 开关，之后每次问答自动触发

**用户看到的变化**：

- 问答流式输出结束后，消息下方出现"评估中..."指示器
- 数秒后（评估在后台运行，不阻塞下一条问题），指示器更新为三项评分：
  - **F（Faithfulness）**：答案是否忠实于检索到的段落，不凭空捏造
  - **AR（Answer Relevancy）**：答案是否切题，回答了用户的问题
  - **CP（Context Precision）**：检索到的段落是否都与问题相关
- 每项评分可展开，查看 AI 评估的理由说明

**Quality 页面**：可查看所有历史问答的评估记录，按会话筛选，了解整体答案质量趋势

---

## 场景二：BEIR 离线检索评估

**使用者**：关注检索质量、对比不同 Embedding 配置效果的开发者 / 研究者

**BEIR 是什么**：一套包含多个领域的标准信息检索数据集（SciFact、FIQA、ArguAna 等），每个数据集包含语料、查询和人工标注的相关性评分。

### 工作流程

**第一步：导入数据集**

```
选择一个 BEIR 数据集（如 SciFact）→ 从 HuggingFace 下载 → 存入本地数据库
```

**第二步：预生成 Embedding**

```
对数据集语料执行批量 Embedding（调用当前 BGE-M3 服务）
→ 以"配置名称"标记（如 bge-m3-v1）
→ 不同配置可独立预生成，互不影响
```

**第三步：运行评估**

```
从数据集中抽取 N 条查询（Sample）
→ 对每条查询做向量检索（在 BEIR 语料内检索，不影响生产知识库）
→ 对比人工标注的相关性，计算标准 IR 指标：
   · nDCG@10：衡量排序质量
   · Recall@10 / @100：衡量召回率
   · MRR@10：衡量第一个相关结果的排名
→ 结果保存，在 Quality 页面可查看历史批次对比
```

**第四步（可选）：注入生产表体验**

```
将 BEIR 数据集文档注入生产知识库
→ 可在 Chat 页面对 BEIR 文档内容提问，体验真实问答效果
→ 不再需要时，一键移除所有注入文档，不影响原有知识库
```

### Quality 页面展示

每次评估批次显示：数据集名称、配置名称、采样数量、各指标得分、运行时间。支持多次批次横向对比，直观看出 Embedding 配置调整带来的效果变化。

---

## 两种评估的定位

|          | 在线评估           | BEIR 离线评估      |
| -------- | ------------------ | ------------------ |
| 评估对象 | 实际问答的答案质量 | 检索系统的检索质量 |
| 数据来源 | 用户真实问题       | 标准基准测试集     |
| 触发方式 | 每次问答自动触发   | 手动执行 CLI 命令  |
| 适合场景 | 监控日常使用质量   | 对比配置、调优检索 |
